## 🎨 UI / UX

### Глобальный ползунок качества на экране сжатия
- **Идея:** Вывести бегунок «Quality» прямо на экран с прогресс-баром, без перехода в настройки.
- **Результат:** Меньше кликов для основного сценария.

### Индивидуальные настройки для каждого файла
- **Идея:** Позволить в рамках одного батча задать одной картинке WebP 80%, а другой — JPEG 90%.
- **Результат:** Профессиональный контроль для веб-мастеров.

### Прогресс-бар для каждого файла
- **Идея:** При загрузке 50 файлов показывать прогресс для каждого отдельного файла, а не только общий.

## 🛠 Технические улучшения

### Ленивая загрузка AVIF-энкодера
- **Зачем:** AVIF WASM модуль весит ~3.5 МБ. Загружать его только если пользователь выберет формат AVIF.
- **Как:** Динамический `import()` в Worker по условию.

### Потоковая архивация (Streaming ZIP)
- **Зачем:** При 100+ файлах `zipSync` удерживает все данные в памяти. Потоковый подход освободит память по мере записи.
- **Как:** Использовать `fflate` streaming API (`Zip` класс вместо `zipSync`).

## 🔧 Расширенный функционал

### Кадрирование (Crop)
Встроенный модальный инструмент — нажать иконку «ножницы» на файле, выделить область и сжать только её.

### Удаление EXIF-метаданных
Галочка «Remove EXIF metadata» в настройках — для приватности при публикации фото.

### Поддержка формата PNG
Добавить `@jsquash/png` для lossless-сжатия PNG без потери качества.

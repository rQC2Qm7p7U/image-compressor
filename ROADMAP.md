# 🚀 Roadmap: Идеи по дальнейшей оптимизации и улучшению

Приложение уже находится в отличном техническом состоянии (Enterprise PWA, WorkerPool, нулевая зависимость от сервера), но нет предела совершенству! Вот список идей для дальнейшего развития:

## ⚡ 1. Технические и архитектурные оптимизации

### ✅ Добавление поддержки AVIF и HEIC (Выполнено в v1.1.0)
- **Зачем:** Формат AVIF сжимает изображения еще на 20-30% эффективнее, чем WebP при одинаковом визуальном качестве. Поддержка HEIC (через `@jsquash/avif` и кастомные декодеры) позволит пользователям iPhone напрямую грузить свои фото со смартфона без предварительной конвертации.
- **Как:** Использованы WASM-модули и `heic2any` прямо на клиентской стороне.

### Аппаратный ресайз через GPU (OffscreenCanvas)
- **Зачем:** Сейчас при ресайзе огромной фотографии (например, 8000x8000 px) до 800px мы отдаем весь этот гигантский массив пикселей в WebAssembly. Это потребляет много оперативной памяти (вплоть до краша вкладки на слабых телефонах).
- **Как:** Использовать встроенный в браузер `OffscreenCanvas`, который задействует видеокарту для мгновенного ресайза картинки до нужных размеров *перед* тем, как отправить ее в WASM для финального сжатия.

### Персистентное хранилище настроек
- **Зачем:** Чтобы пользователю не приходилось каждый раз выставлять любимое качество при новом заходе на сайт.
- **Как:** Обернуть хранилище Zustand в `persist` middleware, которое будет автоматически синхронизировать `settings` с локальным хранилищем браузера (`localStorage` или `IndexedDB`).

## 🎨 2. Развитие UI / UX

### Глобальный ползунок качества прямо в Очереди (Queue)
- **Идея:** Вывести бегунок "Quality (0-100)" прямо на экран со списком загруженных файлов.
- **Результат:** Пользователь может двигать ползунок и в реальном времени (в фоне) наблюдать, как меняется итоговый вес всех файлов, не переходя туда-сюда между вкладками.

### Индивидуальные настройки для каждого файла
- **Идея:** Позволить в рамках одной загруженной партии (batch) задать одной картинке формат WebP 80%, а другой — JPEG 90%. 
- **Результат:** Разблокировка профессионального уровня контроля для сложных задач веб-мастеров.

### Индикаторы прогресса (Progress Bars)
- **Идея:** Если загружено 50 файлов, показывать визуальный прогресс-бар для *каждого* отдельного файла в очереди, вместо одного глобального спинеры, чтобы интерфейс чувствовался более «живым».

## 🛠 3. Расширенный функционал

### Легкое кадрирование (Cropper)
Встроенный модальный `Crop`-инструмент: если пользователю не нужна вся картинка, он может нажать иконку "Ножницы" на файле в очереди, выделить нужный квадрат и сжать только его.

### Удаление EXIF-метаданных по галочке
Умные камеры записывают в фото геопозицию (где было сделано фото) и данные смартфона. Для приватности стоит добавить опцию "Remove EXIF metadata" (Удалить метаданные) в раздел Settings.

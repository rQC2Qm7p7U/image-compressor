# Image Compressor Pro (PWA)

Приложение для пакетного сжатия изображений с поддержкой оффлайн-режима. Работает прямо в браузере.

## Системные требования

Для запуска и разработки вам потребуется **Node.js** (версии 18 или выше).
Скачать можно с официального сайта: [nodejs.org](https://nodejs.org)

## Установка и запуск (macOS / Windows)

Откройте терминал (Terminal на macOS или PowerShell/CMD на Windows) в папке с проектом и выполните следующие команды:

### 1. Установка зависимостей
```bash
npm install
```

### 2. Запуск в режиме разработки
Для локального запуска и проверки работы:
```bash
npm run dev
```
После запуска откройте ссылку, которая появится в терминале (обычно `http://localhost:5173`).

### 3. Сборка и запуск Production-версии (PWA)
Чтобы протестировать работу Service Worker (оффлайн режим) и максимумальной производительности:

```bash
npm run build
npm run preview
```
Откройте ссылку `http://localhost:4173`.

## Использование PWA

### Установка приложения
1. Откройте приложение в браузере **Google Chrome** или **Edge**.
2. В правой части адресной строки появится иконка "Установить" (Install).
3. Нажмите на неё, чтобы установить приложение в систему.
   - **macOS**: Приложение появится в папке `Applications` (Программы).
   - **Windows**: Приложение появится в меню Пуск и на рабочем столе.

### Оффлайн режим
После установки или первого посещения, приложение будет работать без интернета. Все скрипты и стили кэшируются.

### Сохранение файлов
- **Скачать ZIP**: Работает во всех браузерах.
- **Сохранить в папку**: Работает только в Chrome/Edge/Opera на ПК (использует File System Access API).

## Структура проекта
- `src/workers` — Web Worker для сжатия изображений в фоновом потоке.
- `src/views` — Основные экраны интерфейса.
- `src/lib/exportUtils.ts` — Логика экспорта (ZIP, сохранение в папку).

## Возможные проблемы
- **Ошибка "File System Access API not supported"**: Вы используете браузер, который не поддерживает прямой доступ к файловой системе (например, Firefox или Safari). Используйте Chrome или Edge.
- **Черный фон у PNG**: Текущая версия (Stage 1) использует Canvas для сжатия. При конвертации прозрачного PNG в JPEG фон становится черным. Для сохранения прозрачности выбирайте формат **WebP**.
